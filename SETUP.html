Open Store

Important: Square Configuration Required

Action Required: Before the payment system will work, you must add your Square Location ID to the config.js file.
1 Get your Square Location ID:
  • Log in to your Square Dashboard
  • Go to Account & Settings > Locations
  • Click on your location and copy the Location ID
2 Update config.js:
  • Open the config.js file in a text editor
  • Replace REPLACE_WITH_YOUR_LOCATION_ID with your actual Location ID
  • Save the file

Running Locally

Option 1: Python Server (Recommended)

1 Open a terminal in this directory and run:
python server.py

Or on some systems:

python3 server.py

The server will automatically open your browser to http://localhost:8000

Option 2: Just Open in Browser

1 Simply double-click index.html to open it in your default browser.
Note: Some browsers may restrict certain features when opening files directly. Using a local server is recommended.

Going Live (Production)

Important: Before going live, you MUST implement a backend server to process payments securely. The current implementation shows how payments work but requires a server-side component.
1 Switch to Production Credentials:
  • Open config.js
  • Comment out the sandbox configuration
  • Uncomment the production configuration
  • Add your production Location ID
2 Update Square SDK URL:
  • Open checkout.html
  • Change the script source from sandbox to production:
https://web.squarecdn.com/v1/square.js
3 Implement Backend Server:
  • See README.md for complete backend setup instructions
  • Never process payments directly from the frontend
  • Use a backend server to communicate with Square's API
4 Deploy to Hosting:
  • Deploy frontend to any static hosting (Netlify, Vercel, GitHub Pages)
  • Deploy backend to a server (Heroku, AWS, DigitalOcean)
  • Ensure HTTPS is enabled (required by Square)
